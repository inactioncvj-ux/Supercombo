# Tesla 공식 API / SDK 타당성 스터디

작성일: 2026-03-10

## 목적
한국 고속도로 환경에서 테슬라 차량이 **DC콤보 충전소를 경유**할 때,
- 현재 배터리/차량 상태를 반영한 경로 추천
- 도착 예상 잔량 계산
- 충전 전략 추천
- 가능하다면 프리히팅(배터리 프리컨디셔닝) 보조 또는 자동화
를 구현할 수 있는지 **Tesla 공식 API / 공식 SDK 기준으로 검토**한다.

---

## 요약 결론

### 결론 한 줄
**Tesla 공식 API/공식 SDK로 한국형 충전 경로 최적화 앱은 충분히 만들 수 있다.**
다만, **비슈퍼차저 DC콤보에 대해 외부 앱이 배터리 프리히팅을 자동 강제하는 기능은 현재 공개 자료 기준으로 불확실하거나 어려운 쪽**이다.

### 제품 정의 추천
초기 제품은 다음처럼 정의하는 것이 안전하다.

> 한국 테슬라 운전자를 위한 장거리 충전 최적화 앱

프리히팅은 핵심 차별점 후보이지만, **제품의 중심 약속**으로 걸기보다는
**프리히팅 타이밍 안내 / 목적지 설정 유도 / 충전 전략 보조**로 시작하는 편이 현실적이다.

## 조사 근거
이번 스터디는 아래 공개 자료를 바탕으로 정리했다.

### 1) Tesla 공식 공개 저장소: vehicle-command
공개 저장소 README 및 명령 목록/코드 기준으로 확인.

핵심 내용:
- Tesla는 **end-to-end command authentication** 구조를 제공한다.
- 서버가 차량에 명령을 전달하려면 **OAuth token**이 필요하다.
- 차량이 명령을 수락하려면 애플리케이션의 **public key가 차량 keychain에 등록**되어 있어야 한다.
- 공식 SDK는 충전/공조/차량 제어용 명령 세트를 제공한다.

README에서 확인된 핵심 설명:
- “vehicle functions, such as climate control and charging”
- HTTP proxy를 통해 REST API 스타일 명령 전송 가능
- 기존 Owner API는 점차 종료되고, newer vehicles는 command authentication 기반으로 이동

### 2) 공식 SDK 명령 목록(`cmd/tesla-control/commands.go`)
확인된 주요 명령:
- `climate-on`
- `climate-off`
- `climate-set-temp`
- `charging-set-limit`
- `charging-set-amps`
- `charging-start`
- `charging-stop`
- `charge-port-open`
- `charge-port-close`
- `charging-schedule-add`
- `charging-schedule-remove`
- `precondition-schedule-add`
- `precondition-schedule-remove`
- `state`
- `wake`
- 기타 락/트렁크/시트히터/미디어/센트리 모드 등

### 3) 공식 SDK 코드(`pkg/vehicle/charge.go`, `pkg/vehicle/climate.go`)
코드에서 직접 확인한 것:
- 충전 한도 변경
- 충전 시작/중지
- 충전 전류 설정
- charge schedule 추가/삭제
- precondition schedule 추가/삭제
- climate on/off
- 온도 설정

중요한 점:
- 공개 코드/명령 목록 기준으로는
  - **“지금 배터리 프리히팅 즉시 시작”**
  - **“특정 충전소를 목적지로 전송”**
  - **“이 위치를 급속충전소로 인식시켜라”**
  같은 명령은 확인하지 못했다.

## 기능별 판정

## 1. 구현 가능성이 높은 기능

### A. 차량 상태 읽기
판정: **높음**

근거:
- 공식 SDK/공식 생태계에서 vehicle state/category 접근 및 state 명령이 존재
- Fleet API / telemetry 구조도 존재

앱에 활용 가능한 정보(예상):
- 현재 SoC
- 충전 상태
- 차량 wake 상태
- 위치/주행 관련 일부 상태
- 차종 기반 분기

활용 시나리오:
- 현재 SoC 기반 충전 전략 계산
- 출발지/목적지 기준 추천 경유지 계산
- 도착 예상 잔량 계산

### B. 한국형 충전 경로 추천
판정: **매우 높음**

이 기능은 Tesla API가 아니라 **우리 쪽 경로/에너지 모델링**의 문제다.

필요 요소:
- 출발지/목적지
- 현재 SoC
- 차종
- 외기온도
- 고속도로 경로
- 한국 DC콤보 충전소 DB

출력 가능 항목:
- 어느 휴게소를 경유할지
- 충전소 도착 예상 SoC
- 몇 분 충전해야 하는지
- 최종 도착 예상 SoC

### C. 충전 관련 일부 제어
판정: **높음**

공식 SDK에서 직접 확인된 것:
- charging start / stop
- charge limit 설정
- charging amps 설정
- charge schedule 설정
- charge port open/close

비고:
- 장거리 충전 경로 앱의 핵심은 아니지만,
  나중에 프리미엄/고급 기능으로 엮을 수 있다.

---

## 2. 구현 가능하지만 검증이 더 필요한 기능

### D. 고정밀 도착 잔량 예측
판정: **중상**

가능한 이유:
- 차량 상태 + 외부 경로 + 온도 + 고도 데이터를 합쳐 모델링 가능
- telemetry를 활용하면 고도화 여지가 큼

어려운 이유:
- 속도
- 외기온도
- 공조 사용
- 고도 변화
- 교통 정체
- 풍향/풍속
- 차종별 전비 편차

결론:
- 구현은 가능
- 하지만 정확도는 별도 모델링/학습 품질에 좌우됨

### E. 차량 내 목적지 연동
판정: **불확실**

확인한 내용:
- 공개 command 목록에서 navigation/destination 계열 명령을 직접 확인하지 못함
- 공개 charge/climate 관련 코드에서도 목적지 전송 로직은 보이지 않음

의미:
- 공식 API에 있을 수도 있으나, 현재 공개 자료만으로는 확인 못 함
- 상용 개발자가 일반적으로 쓰는 범위가 아닐 가능성 있음

필요한 후속 검증:
1. developer.tesla.com의 Fleet API endpoint 문서 직접 확인
2. destination / navigation / share-to-car 유사 엔드포인트 유무 확인
3. 목적지 전송 시 차량이 일반 목적지인지 충전 목적지인지 구분 가능한지 확인

---

## 3. 가장 위험하거나 불확실한 기능

### F. 비슈퍼차저 DC콤보 자동 프리히팅
판정: **낮음 ~ 불확실**

우리가 원하는 기능:
- 한국 고속도로 휴게소 DC콤보를 경유지로 선택
- 외부 앱이 차량에 충전 목적지를 알려줌
- 차량이 슈퍼차저 가는 것처럼 배터리 프리히팅(프리컨디셔닝)을 자동 수행

현재 공개 자료 기준 문제점:
- “배터리 프리히팅 즉시 시작” 명령을 찾지 못함
- “특정 충전소를 충전 목적지로 인식시켜라” 명령을 찾지 못함
- `precondition-schedule-add`는 존재하지만,
  이름상/구조상 **출발 시간 기반 프리컨디셔닝 스케줄**에 더 가까워 보임
  (주행 중 DC 급속충전 전 배터리 예열 강제와는 성격이 다를 가능성이 큼)

중간 판단:
> 현재 공개된 Tesla 공식 SDK/API 자료만으로는,
> 외부 상용 앱이 임의의 한국 DC콤보 충전소에 대해 배터리 프리히팅을 자동 강제할 수 있다고 보기 어렵다.

## 제품 전략상 해석

### 하면 안 되는 약속
- “한국 DC콤보에서도 프리히팅을 자동 구현하는 앱”

이 문장은 현재 너무 위험하다.
왜냐하면 Tesla 공식 공개 범위에서 그 기능이 보장되는지 확인되지 않았기 때문이다.

### 안전한 약속
- “한국 테슬라 장거리 충전 최적화 앱”
- “DC콤보 경유 전략과 프리히팅 타이밍을 함께 제안하는 앱”
- “한국 고속도로 환경에 맞춘 테슬라 충전 라우터”

이건 현실적이고 상용화하기에도 안전하다.

- 현재 SoC 읽기
- 차종 반영

- 충전 시간 추천

### 유료화 핵심 기능
- 외기온도 반영
- 고도 반영
- 차종별 전비 보정
- 혼잡도/충전기 신뢰도 반영
- 사용자의 실제 주행 로그 기반 개인화

- “도착 20~30분 전 차량 내 네비에 충전소를 지정하세요” 가이드
- “현재 배터리 온도/외기온도 조건상 프리히팅 유도가 필요한 구간” 안내
- 충전소별 프리히팅 체감 반응에 대한 경험 기반 추천(추후 커뮤니티/데이터 기반)

## 추가로 확인해야 할 항목

### 1. Tesla 개발자 포털 직접 검증
이번 스터디에서는 developer.tesla.com 페이지 fetch가 차단되어,
공개 GitHub 저장소 중심으로 판단했다.
따라서 다음 항목은 **직접 문서 열람**으로 재확인 필요:

- Fleet API vehicle data endpoint 상세
- Fleet API vehicle commands endpoint 상세
- destination/navigation/share-to-car 관련 endpoint 유무
- 상업적 사용 및 rate limit 정책
- OAuth / third-party token 발급 플로우

### 2. 실제 차량 테스트 필요
문서만으로 확정하기 어려운 항목:
- 목적지 전송 시 차량 반응
- precondition schedule이 실제로 어떤 동작을 의미하는지
- 급속충전 전 배터리 컨디셔닝과 공조 preconditioning의 차이

### 3. 한국 데이터 소스 정리 필요
- 고속도로 휴게소 DC콤보 목록
- 충전기 운영 상태
- 출력/혼잡도/신뢰도
- 고속도로 진출입 우회 비용

## 최종 판단

### Yes
- Tesla 공식 API 기반의 상용 앱 제작은 가능성이 높다.
- 특히 **장거리 충전 경로 최적화 앱**은 충분히 현실적이다.

### Not Yet Proven
- 한국의 비슈퍼차저 DC콤보 충전소에 대해
  외부 상용 앱이 배터리 프리히팅을 자동 강제할 수 있는지는
  현재 공개 자료만으로 증명되지 않았다.

### 따라서
이 프로젝트는 아래 포지션으로 시작하는 것이 맞다.

> **한국형 테슬라 충전 최적화 앱 + 프리히팅 보조 UX**

이 포지션이면 현실적이고, 유료화에도 맞고, 나중에 공식 API 범위가 확인되면 자동화 기능을 확장할 수 있다.
