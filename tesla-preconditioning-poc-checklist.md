# Tesla 프리컨디셔닝 / DC콤보 연동 PoC 체크리스트

작성일: 2026-03-10

## 목적
Tesla 공식 API / 공식 SDK 범위에서,
**비슈퍼차저 DC콤보 충전소에 대해 자동 또는 반자동 프리히팅(배터리 프리컨디셔닝)이 가능한지** 실차 기준으로 검증한다.

이 문서는 문서 해석이 아니라 **실험 설계 문서**다.

---

## 검증하고 싶은 핵심 질문

### Q1.
`SetPreconditioningMax`는 실제로 무엇을 켜는가?
- 단순 cabin preconditioning인가?
- 배터리 컨디셔닝까지 포함하는가?
- 급속충전 전 프리히팅과 동일한가?

### Q2.
`precondition-schedule-add`는 실제로 어떤 동작을 유발하는가?
- 출발 전 시간 기반 준비인가?
- 특정 위치에서의 컨디셔닝인가?
- 주행 중 충전 목적지 접근 시 활용 가능한가?

### Q3.
외부 앱이 임의 위치(한국 휴게소 DC콤보)를 차량에 전달했을 때,
차량이 이를 **충전 목적지**로 인식하는 공식 경로가 있는가?

### Q4.
비슈퍼차저 DC콤보에 대해 슈퍼차저와 유사한 배터리 프리히팅이 가능한가?
- 완전 자동
- 반자동
- 불가
중 어느 수준인가?

## 준비물

### 차량
- Tesla 차량 1대 이상
- 가능하면 테스트 대상 차종 명시
  - Model Y LR
  - Model 3 LR
  - 기타

### 계정/권한
- Tesla 개발자 계정
- Fleet API access token
- vehicle-command용 private key / public key enrollment 완료
- 테스트 차량에 third-party app key 등록 완료

### 도구
- Tesla 공식 `vehicle-command` SDK 또는 `tesla-control`
- 요청/응답 로깅 환경
- 필요 시 proxy server
- 주행 로그 기록 수단
- 가능하면 OBD / Scan My Tesla 계열 보조 측정 수단
  - 배터리 온도
  - pack power
  - charging preheat 관련 지표

### 환경 조건
- 외기온도 낮을수록 테스트 유리
  - 예: 0~10℃
- 장거리 주행 또는 충분한 냉간 상태
- 도착 직전 DC 급속충전 속도 비교 가능한 환경

## 실험 1 — `SetPreconditioningMax` 단독 효과 확인

### 목적
`SetPreconditioningMax`가 단순 cabin preconditioning인지,
배터리 관련 컨디셔닝까지 포함하는지 확인한다.

### 절차
1. 차량을 충분히 식힌다.
2. 차량 주차 상태에서 baseline 측정
   - 외기온도
   - 배터리 상태
   - 가능하면 배터리 온도
   - 실내 공조 상태
3. 공식 SDK/명령으로 `SetPreconditioningMax(on=true)` 실행
4. 차량 앱/차량 화면/로그/보조 계측에서 변화 확인

### 관찰 포인트
- HVAC만 켜지는가?
- 실내 온도 변화만 있는가?
- 배터리 히팅/컨디셔닝 징후가 있는가?
- 에너지 소비 패턴이 급속충전 대비 프리히팅과 유사한가?

### 판정
- **A:** cabin only
- **B:** cabin + battery 가능성 있음
- **C:** 급속충전 전 배터리 프리히팅과 매우 유사

---

## 실험 2 — `precondition-schedule-add` 동작 의미 확인

### 목적
위치+시간 기반 precondition schedule이 실제로 뭘 의미하는지 확인한다.

### 절차
1. 임의 좌표(예: 집)로 schedule 추가
2. 같은 날/다른 날에 반복 테스트
3. 조건:
   - 차량이 해당 위치에 있을 때
   - 해당 위치에 없을 때
4. preconditioning이 어떤 시점에 켜지는지 기록

### 관찰 포인트
- schedule이 출발 전 준비로 작동하는가?
- 위치 매칭이 필요한가?
- 차량이 이동 중일 때도 작동하는가?
- 충전 목적지 접근과는 관계가 있는가?

### 판정 기준
- **출발 준비 스케줄용**인지
- **위치 기반 도착/접근 이벤트형**인지
- **주행 중 충전 유도용으로 전용 가능**한지

---

## 실험 3 — 슈퍼차저 목적지와 비교 테스트

### 목적
차량이 공식적으로 충전 전 프리히팅을 수행하는 대표 시나리오와 비교 기준을 만든다.

### 절차
1. 차량 내 네비에서 슈퍼차저 목적지 설정
2. 주행 중 프리히팅 시작 시점 기록
3. 가능하면 배터리 온도/전력 소비/log 기록

### 관찰 포인트
- 프리히팅 시작까지 걸리는 시간
- UI에서 보이는 프리컨디셔닝 문구
- 충전 도착 시 초기 charging power

### 왜 필요한가
이게 기준선이다.
나중에 DC콤보 실험과 비교해서,
“비슷하게 동작했는지 / 전혀 다른지”를 판단할 수 있다.

---

## 실험 4 — 한국 DC콤보 목적지 접근 시 자동 반응 확인

### 목적
슈퍼차저가 아닌 한국 DC콤보 충전소를 향해 갈 때,
차량이 자동 프리히팅 반응을 하는지 확인한다.

### 절차
1. 휴게소 DC콤보 좌표 선정
2. 차량 내 기본 네비 또는 가능한 공식 연동 방식으로 목적지 설정
3. 주행 중 프리컨디셔닝 반응 관찰

### 관찰 포인트
- 차량 UI에 charging-related preconditioning 표시가 있는가?
- 배터리 온도 상승 패턴이 있는가?
- 도착 직후 charging speed가 cold-gated 상태보다 높은가?

### 판정
- **자동 반응 있음**
- **일반 목적지처럼만 처리됨**
- **전혀 반응 없음**

---

## 실험 5 — 외부 명령 + DC콤보 접근 조합 테스트

### 목적
공식 SDK 명령과 DC콤보 경로를 조합했을 때 실용적 우회가 가능한지 본다.

### 후보 조합
1. `SetPreconditioningMax(on=true)` + DC콤보 접근
2. `precondition-schedule-add` + 휴게소 좌표
3. 외부 앱 알림 + 차량 내 목적지 지정

### 확인 포인트
- 실사용자가 체감 가능한 수준의 charging speed 개선이 있는가?
- 운전자 개입이 적은가?
- 반복 재현성이 있는가?

### 목표
완전 자동이 안 되더라도,
**반자동 UX로 제품화 가능한 수준인지** 판단한다.

## 사업 판단 기준

### 제품화 가능
다음 중 하나라도 성립하면 제품 차별화 가능성이 높다.
- 외부 명령으로 배터리 컨디셔닝이 재현됨
- DC콤보 도착 전 charging speed 개선이 반복 확인됨
- 완전 자동은 아니어도 반자동 UX로 충분히 가치가 있음

### 제품 핵심 약속으로 쓰면 안 되는 경우
다음이면 자동 프리히팅을 마케팅 핵심으로 걸면 안 된다.
- cabin preconditioning만 확인됨
- 배터리 관련 효과가 불명확함
- 특정 조건에서만 우연히 반응함
- 반복 재현이 안 됨

## 현재 잠정 판단
문서만으로는 다음이 가장 정확하다.

- Tesla 공식 SDK에는 preconditioning 관련 명령이 존재한다.
- 하지만 그 명령이 **비슈퍼차저 DC콤보 급속충전 전 배터리 프리히팅**을 의미하는지는 아직 입증되지 않았다.
- 따라서 실차 PoC 없이는 제품 핵심 약속으로 쓰면 위험하다.
