# Supercombo 프리컨디셔닝 실행 구조 재설계

작성일: 2026-03-10

## 배경
초기 가설은 고속도로 휴게소 DC콤보를 차량이 공식 충전 목적지로 인식하게 만들어,
차량 기본 로직으로 프리컨디셔닝을 유도하는 것이었다.

하지만 실차 기준 현재 확인된 사실은 다음과 같다.
- **고속도로 휴게소 DC콤보는 차량에서 공식 충전 목적지로 잡히지 않는다.**

따라서 프리컨디셔닝 구조를 다시 설계해야 한다.

---

## 목표
Supercombo는 차량의 공식 충전 목적지 인식에 기대지 않고도,
최적 충전 경유지에 맞춰 **프리컨디셔닝을 적절한 시점에 실행**할 수 있어야 한다.

---

## 핵심 구조

### 기존 구조
1. 앱이 충전소 추천
2. 차량이 충전 목적지로 인식
3. 차량이 자동 프리컨디셔닝

### 새로운 구조
1. 앱이 충전소 추천
2. 앱이 도착 시점 / 배터리 상태 / 외기온도 계산
3. 앱이 프리컨디셔닝 시작 시점 결정
4. 앱이 공식 API 또는 사용자 액션으로 프리컨디셔닝 실행

즉 구조의 중심이 차량 내비에서 앱으로 이동한다.

---

## 프리컨디셔닝 실행 레벨

### Level 1 — 자동 실행
가장 이상적인 방식.

조건:
- Tesla 공식 API에서 프리컨디셔닝 관련 실행 명령 가능
- 앱이 적절한 시점에 자동 호출

장점:
- 사용자 경험이 가장 좋음
- 차량 충전 목적지 인식과 무관하게 앱이 주도 가능

리스크:
- 공식 API에서 실제로 가능한지 검증 필요
- 단순 cabin preconditioning인지, 배터리까지 포함하는지 검증 필요

### Level 2 — 반자동 실행
차선책.

구조:
- 앱이 프리컨디셔닝 시작 시점을 계산
- 사용자에게 즉시 실행 버튼 제공
- 사용자는 1탭으로 실행

장점:
- 기술 제약이 있어도 현실적으로 구현 가능
- UX 손실이 크지 않음

리스크:
- 자동 실행보다 경험이 약함

### Level 3 — 강한 유도
가장 약한 방식.

구조:
- 앱이 “지금 프리컨디셔닝하세요”를 알려줌
- 실제 실행은 사용자가 차량/앱에서 직접 함

장점:
- 구현 쉬움

단점:
- Supercombo의 차별성이 약해질 수 있음

---

## 권장 방향
Supercombo는 아래 우선순위로 구현해야 한다.

### 우선순위 1
**Level 1 자동 실행 가능성 검증**
- 공식 API로 프리컨디셔닝 실행 가능한지
- 실행 시 실제 배터리 프리컨디셔닝까지 이어지는지

### 우선순위 2
**Level 2 반자동 UX 설계**
- 자동 실행이 불안정하면 즉시 반자동으로 전환
- 추천 경로와 함께 프리컨디셔닝 실행 버튼 제공

### 우선순위 3
**Level 3 최소 유도**
- 최악의 경우에도 프리컨디셔닝 타이밍 안내는 제공

---

## 입력 변수
프리컨디셔닝 실행 판단에는 최소한 아래가 필요하다.

- 현재 SoC
- 현재 위치
- 추천 충전소 위치
- 도착 예상 시각
- 외기온도
- 차량 모델
- 주행 속도 추정

---

## 출력
앱은 최종적으로 아래 중 하나를 해야 한다.

1. **프리컨디셔닝 자동 실행**
2. **프리컨디셔닝 1탭 실행**
3. **프리컨디셔닝 즉시 유도 알림**

---

## UX 흐름

1. 사용자가 목적지 입력
2. Supercombo가 고속도로 휴게소 DC콤보 추천
3. 추천 결과에 충전소 도착 SoC, 충전 시간, 최종 도착 SoC 표시
4. 앱이 프리컨디셔닝 시작 예상 시점 계산
5. 시점 도달 시
   - 자동 실행 가능하면 실행
   - 아니면 실행 버튼 또는 강한 알림 제공

---

## 중요한 제품 해석
이 구조는 Supercombo를 단순 경로 추천 앱이 아니라,
**충전 판단 + 차량 준비 행동까지 책임지는 앱**으로 바꾼다.

즉 Supercombo의 차별점은
- 어디서 충전할지 추천하는 것뿐 아니라
- **그 충전에 맞게 차량을 준비시키는 것**에 있다.

---

## 최종 결론
현재 한국 환경에서는 휴게소 DC콤보를 차량이 공식 충전 목적지로 인식하지 않으므로,
Supercombo는 **앱 주도 프리컨디셔닝 구조**로 가야 한다.

즉 핵심은:
- 최적 충전 경유지를 선정하고
- 그 경유지 기준으로
- 적절한 시점에
- 앱이 프리컨디셔닝을 실행하거나 실행하게 만드는 것

이다.
