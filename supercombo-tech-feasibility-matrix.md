# Supercombo 기술 가능/불가능 표

작성일: 2026-03-10

## 목적
Supercombo의 핵심 요구사항을 기준으로,
현재까지 파악한 기술적 가능 범위와 불확실성을 정리한다.

---

## 범례
- **가능**: 현재 기술적으로 구현 가능성이 높음
- **조건부 가능**: 외부 API/실데이터/추가 검증 필요
- **불확실**: 문서상 단서가 부족하거나 실차 검증 필요
- **위험**: 제품의 핵심 약속으로 바로 걸기 어려움

## 가장 확실히 가능한 것

### 1. 충전 판단 엔진
- 현재 SoC 기반 추천
- 경유 충전소 추천
- 도착 잔량 계산
- 충전 시간 제안
- 대안 경로 제시

이 부분은 Supercombo의 몸통이 될 수 있다.

### 2. 고속도로 DC콤보 데이터 정제
- 공공데이터 기반 수집
- 정규화
- 휴게소명/방향/고속도로명 추론
- 제품용 데이터셋 생성

이미 파이프라인 골격이 존재한다.

### 3. 테슬라 공식 앱에 가까운 UX
- 지도 위 경로
- 충전 경유지 카드
- 출발/도착 잔량
- 차로 전송 버튼

이건 기술적 제약보다 설계/구현 문제다.

---

## 가장 중요한 불확실성

### 1. 차로 전송하기
핵심 질문:
- 서드파티 앱이 목적지/경유지를 차량으로 전송할 수 있는가?
- 단순 목적지 전송인지, 전체 경로 전송인지?

이게 되면 UX가 크게 올라간다.

### 2. 충전소로 인식
핵심 질문:
- 전송된 경유 충전소를 차량이 일반 목적지가 아니라 **충전소**로 인식하는가?

이게 안 되면 Supercombo는 좋은 충전 최적화 앱이지만,
봉봉이 원하는 “슈차처럼” 경험엔 못 닿는다.

### 3. 실제 프리컨디셔닝 발동
핵심 질문:
- 충전소 인식 이후 실제 배터리 프리컨디셔닝이 발동되는가?

이건 Supercombo의 킬러 포인트이자 가장 큰 리스크다.

---

## 제품적으로 해석하면

### 가능성이 높은 제품
**한국형 테슬라 장거리 충전 최적화 앱**

### 킬러 포인트까지 달성한 제품
**고속도로 DC콤보를 슈퍼차저처럼 쓰게 만드는 앱**

둘의 차이는 결국 아래 3개에 달려 있다.
- 차로 전송하기
- 충전 목적지 인식
- 실제 프리컨디셔닝 발동

---

## 현재 권장 전략

### 1단계
확실히 가능한 것부터 만든다.
- 공식 API 연동
- 충전 판단 엔진
- 전국 고속도로 구조
- 단일 경유 추천
- 실시간 충전현황 연동 시도

### 2단계
UX 핵심을 검증한다.
- 차로 전송하기 가능성 확인
- 결과 화면을 테슬라 공식 앱처럼 구현

### 3단계
킬러 포인트를 검증한다.
- 충전소 인식 여부
- 실제 프리컨디셔닝 발동 여부

---

## 최종 판단
Supercombo는 현재 기준으로도 충분히 가치 있는 제품이 될 수 있다.
다만 봉봉이 정의한 **진짜 성공**은 아래까지 도달해야 한다.

1. 차로 전체 경로 전송
2. 차량의 충전소 인식
3. 실제 프리컨디셔닝 발동
4. 사용자가 “테슬라 기본보다 낫다”고 느낌

즉,
**앱 자체는 만들 수 있다.**
하지만 **Supercombo의 킬러 포인트는 반드시 별도 기술 검증이 필요하다.**
